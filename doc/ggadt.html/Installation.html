2 Installation

2.1 System requirements

GGADT should install easily on most Unix-based systems (e.g. Linux, Mac) by following the standard GNU install procedures, described in INSTALL. 

     ./configure [--enable-openmp] ...
     make
     make install

2.2 Package dependencies

  • FFTW 3. Available at http://www.fftw.org/.
    • Further requirements for installing ggadt-openmp: you must have the fftw3_omp libraries installed. To do this, specify the --enable-openmp option when running configure for FFTW3.
    • Location of the FFTW3 headers: In order for GGADT to find the Fortran headers for FFTW3, they must be in the standard location (/usr/local/include or usr/include) or in $prefix/include where $prefix is defined by doing ./configure --prefix=<prefix> at the configuration step for ggadt.
  • Suitable C and Fortran compilers: GGADT was developed using the GNU compiler suite (version 4.9) throughout, but the configure script will try to work with most common compilers. Your Fortran compiler must be able to compile Fortran 2003 code.
    • OpenMP: Further requirements for installing ggadt-openmp: For the OpenMP version of GGADT, you must have C and Fortran compilers that are compatible with OpenMP version 4.0. This version of GGADT was tested with the GNU C and Fortran compilers (gcc, gfortran) version 4.9.
  • Mac users
    • Install the latest edition of XCode command-line tools. As of Mac OSX v10.9.1, the command-line tools are not included in XCode automatically, so you have to do this on your own.
    • Warning about the “gcc" compiler included in the XCode command line tools. As I understand it, this is actually a front-end to a clang compiler, and will cause problems if you try to compile ggadt-openmp. The gcc version is also quite dated and might cause other problems. Your best bet is to either use another compiler (try the Intel compiler suite) or to download later versions of the GNU compilers. We recommend obtaining later versions of GNU compilers, which you can find here: http://hpc.sourceforge.net/.
    • MacPorts – In order to avoid confusing the configuration script, we recommend you avoid using MacPorts, as they have their own directory structure and program names. It might still be possible to work with MacPorts as long as all of the MacPort program installations that you would need (i.e. gcc, gfortran) have symlinks to the locations that GNU Autoconf expects them to be. I.e., if you have a MacPort installation of gcc called gcc-4.8 in some local directory, you should have a symlink to the binary gcc-4.8 file in /usr/bin, with a symlink name of gcc. We haven't tried this, and thus we offer no gaurantee that this would work.
  • (Optional) Python 2.7 or later for some of the included scripts.
  • (Optional matplotlib python library if you wish to use the included plotting scripts.
